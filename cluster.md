# GAIPAT 切片级无监督聚类方案 (Yeo-Johnson 修正版)

## 1. 核心目标与硬性约束

* **数据定义明确**：输入特征严格基于 `deviation_cm`。该值为视线到目标多边形的最短直线距离（点在内部或边缘上为 `0`，在外部为 `>0` 的绝对厘米数）。
* **聚类单位**：切片级（Slice-level），即对每个时序变长的 `.csv` 或 `.jsonl` 切片提取固定长度的特征向量。
* **事件隔离**：`grasp`（抓取）和 `release`（放置）事件必须**分别**建立独立的聚类模型，不可混合。
* **标签约束（金标准）**：所有样本（成功与失败）共同参与聚类。但聚类完成后，**原始失败样本（`label==0`）必须被硬性覆写为“分心 / 非专注”**。聚类的作用仅限于细分成功样本的内部状态，或用于事后解释失败样本的错误模式。

---

## 2. 数据预处理与特征工程

鉴于边缘距离数据存在严重的“长尾效应”，我们将采用先基于原始数据提取物理特征，再在数据集层面利用 Yeo-Johnson 算法进行自适应高斯化和标准化的策略。

### 2.1 核心特征集 (Top 10)

直接基于原始的最短距离序列 $d_{raw}$ 提取以下 10 个固定长度特征：

* **距离水平特征**：
* `distance_median`: 距离中位数
* `distance_q10`: 第 10 百分位数（反映最专注时的状态）
* `distance_q90`: 第 90 百分位数（反映最游离时的状态）


* **距离波动特征**：
* `distance_std`: 距离标准差


* **距离趋势特征**：
* `distance_slope`: 距离对时间的线性回归斜率（负值代表视线正在收敛）


* **平稳性特征**：
* `diff_abs_mean`: 一阶差分绝对值的均值（反映视线跳跃程度）
* `distance_lag1_autocorr`: 滞后 1 阶的自相关系数


* **持续贴近目标特征** (设定绝对物理容错阈值 $\tau = 2.0$ cm)：
* `near_ratio_tau`: $d_{raw} < 2.0$ 的帧数量占比
* `longest_near_run_ratio`: 最长连续满足 $d_{raw} < 2.0$ 的序列长度占比
* `time_to_first_near_ratio`: 首次满足 $d_{raw} < 2.0$ 所花费的时间占比



### 2.2 Yeo-Johnson 变换与标准化 (核心改进)

将所有切片的上述 10 个特征提取完毕，构建成特征矩阵后，在输入聚类算法前执行 **Yeo-Johnson 变换**。

**算法优势**：

1. **完美处理零值**：克服了传统 Box-Cox 和对数变换无法处理 $d_{raw} = 0$（视线完全落在积木内）的数学缺陷。
2. **自适应正态化**：算法会为每一个特征列自动寻找最优的 $\lambda$ 参数，将长尾偏态分布“揉捏”成最适合 GMM 算法的高斯（正态）分布。
3. **内建标准化**：在转换分布的同时自动完成零均值、单位方差的处理，彻底替代原方案中的 `log1p` 和 `RobustScaler`。

**代码实现参考**：

```python
from sklearn.preprocessing import PowerTransformer

# 启用 yeo-johnson 并在变换后自动标准化
pt = PowerTransformer(method='yeo-johnson', standardize=True)
# df_features 为提取好的包含所有切片特征的 DataFrame
X_transformed = pt.fit_transform(df_features) 

```

## 3. 算法选择与执行管线

**推荐核心算法：GMM (Gaussian Mixture Model)**
GMM 的本质是拟合多个高斯分布的混合体。由于前置的 Yeo-Johnson 变换已经将特征空间优化为近似高斯分布，GMM 将在此数据上发挥出极佳的聚类性能，且能输出软概率（Soft Probabilities）。

**执行步骤：**

1. **数据读取与提取**：读取第三步的 `.jsonl` 切片，提取 `deviation_cm`（即 $d_{raw}$），并为每个切片计算出上述 10 个统计和时序特征。
2. **构建特征表**：生成一张未经缩放的原始特征表 `cluster_raw_features.csv`。
3. **分离事件**：将特征表按 `event == grasp` 和 `event == release` 拆分。
4. **特征变换与聚类**：
* 对拆分后的特征矩阵分别应用 `PowerTransformer(method='yeo-johnson')`。
* 使用变换后的特征矩阵独立拟合 GMM 模型。建议初始设定簇数量 `k=3` 或 `k=4` 进行测试。


5. **硬规则覆盖**：将聚类输出的簇标签与原始操作标签 `label` 联表，将所有 `label == 0` 的样本强制覆写标记为 `Distracted`。

## 4. 预期簇语义与评估指南

评估聚类效果时，不要过度依赖 Silhouette Score 等纯数学指标。**最重要的评估标准是“簇的平均距离曲线是否具有物理可解释性”**。

在合理的聚类结果中，成功样本（`label == 1`）通常会被划分为以下几类典型模式：

1. **Focused (专注 / 稳定)**
* **特征表现**：`distance_median` 接近 0，`near_ratio_tau` 极高，波动极小。
* **物理意义**：视线精准落在多边形内部或紧贴边缘（偏差 $< 2.0$ cm），且动作全过程保持稳定。


2. **Searching (任务相关搜索)**
* **特征表现**：`distance_slope` 为显著负值，初期距离大，后期稳定收敛到 0。
* **物理意义**：起初视线在别处（例如刚做完上一个动作，正在寻找下一个目标），随后快速锁定当前目标并完成操作。


3. **Wandering (游移 / 不稳定)**
* **特征表现**：波动极大，`longest_near_run_ratio` 占比较低。
* **物理意义**：视线在目标与环境之间反复横跳，虽然最终勉强完成了动作，但注意力集中度较低。


4. **Noise (数据噪声)**
* **特征表现**：平稳性特征极差，一阶差分（`diff_abs_mean`）极大。
* **物理意义**：眼动仪置信度低造成的信号剧烈跳变。
