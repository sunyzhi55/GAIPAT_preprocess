### 问题（2）：每个任务、每个受试者的时间片段长度不一致，无法直接求“平均偏差序列”，如何解决？

这是处理时序不均匀数据（Non-uniform Time Series）必然面对的挑战 。由于你整合的 DataFrame 时间戳不连续且间隔不固定，每个人因为思考延迟、动作快慢，导致同一个步骤的采样行数（序列长度）大相径庭。

为了求解出统一的“标准专注模板”**，你需要采用**“时序规整与对齐（Resampling & Alignment）”策略，具体标准执行步骤如下：

#### 步骤一：基于时间进度的归一化与线性插值（Progress-based Resampling）

我们不以“行数”或“绝对时间秒数”为对齐标准，而是引入“任务执行进度百分比（$0\% \sim 100\%$）”的概念，将所有成功片段强制重采样（Resample）到相同的物理长度 $L$（推荐 $L = 100$ 或 $200$ 个点）。

**具体算法逻辑：**


1. 计算该片段内每一行的距离偏差（使用问题 1 的方案二），得到一个变长的一维偏差序列 $D_{raw} = [d_1, d_2, ..., d_N]$。
2. 对应的原始时间戳序列为 $T = [t_1, t_2, ..., t_N]$。我们将时间归一化到百分比进度：$P = \frac{T - t_1}{t_N - t_1}$，此时 $P$ 是一个从 $0.0$ 到 $1.0$ 的递增序列。
3. 设定标准的对齐网格 $P_{std} = \text{np.linspace}(0.0, 1.0, L)$。
4. 利用一维线性插值 `np.interp`，将原始偏差映射到标准网格上，得到长度固定为 $L$ 的标准长度偏差序列 $D_{std}$。

#### 步骤二：计算标准专注模板（Standard Vigilance Template）

当你把所有参与者的**所有成功（`is_success == 1`）片段**都通过上述步骤缩放到固定长度 $L$ 之后，你就拥有了一个对齐的矩阵。

假设一共有 $K$ 个成功的操作片段，把它们堆叠成一个形状为 $(K, L)$ 的二维矩阵。此时，你就可以安全地**跨行求算术平均值**，得到你论文的核心基石——“标准专注模板（长度为 $L$ 的一维向量）” ：

$$\text{Standard\_Template}[j] = \frac{1}{K} \sum_{i=1}^K \text{Matrix}[i, j], \quad \text{for } j = 0, 1, ..., L-1$$

#### 步骤三：利用 DBA 算法（DTW Barycenter Averaging）进行高级修正（AAAI 强推）

* **潜在风险**：直接做算术平均可能会“抹平”时序波形中的特征峰值（例如：受试者总是在进度 $30\%$ 的地方低头找积木，有人早有人晚，直接平均会让这个特征变平）。
* **降维打击做法**：使用 `tslearn` 库中的 `Barycenter` 模块，采用 **DBA（DTW 重心平均）** 算法。它是一种基于动态时间规整（DTW）的时序序列平均方法，专门用来求解一组长度不等、速度不同的时间序列的“真实拓扑均值序列”。

```python
from tslearn.barycenters import dtw_barycenter_averaging

# 假设 success_sequences 是一个 list，里面包含很多一维 numpy 数组（长度可以不等）
# 每个数组代表一个成功任务的原始物理偏差序列
# dtw_barycenter_averaging 会自动处理长度异质性，并迭代求解出标准的重心模板序列
standard_template = dtw_barycenter_averaging(success_sequences, max_iter=50)

```
