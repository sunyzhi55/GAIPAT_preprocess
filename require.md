# GAIPAT 数据集高频多模态同步与特征工程流水线技术规范

## 一、 环境参数与物理尺寸基准 (Environment Constants & Calibration Anchors)

在进行任何空间变换和坐标量纲转换前，必须在代码初始化阶段显式声明以下固定物理常量（源于 `setup/` 元数据规约）：

* **显示器 (Dell P2416D)**：物理尺寸 $52.7 \times 29.6\text{ cm}$，分辨率 $2560 \times 1440\text{ 像素}$。
* **玻璃支撑桌 (Plexiglass Table)**：物理尺寸 $100 \times 50\text{ cm}$。
* **LEGO 装配表面 (LEGO Surface)**：物理尺寸 $76 \times 38\text{ cm}$。
* **预置块剔除规则**：在解析 `setup/instructions_{task}.csv` 时，一般第一行数据（`id == 0`），表示当前对应的 积木id 代表底座或系统预先放置好的静态参照物，受试者无需对其进行抓取或对准操作。代码在构建动态轨迹片段时，应对 `id == 0` 的对应的 积木id 的行进行特异性过滤或仅将其视作静态背景锚点。

## 二、 输出大宽表（Master DataFrame）结构定义与字典规约

### 1. 文件组织架构

脚本必须以受试者为核心单元进行闭环处理。遍历所有样本文件夹后，按照每个 `subject_id` 分别建立独立的存储文件夹，并在其内部针对每个 `task`（如 `car`, `house` 等）单独生成并保存一张处理对齐后的 `csv` 文件。

### 2. 表格主键与时间轴生成逻辑

重构后的数据集将彻底**以高频眼动时间戳（`Timestamp`）作为唯一主键**。离散的动作事件（Events）和系统状态（States）在连续的时间区间内，一律采用前向填充（Forward-fill, `ffill`）的方式动态广播到每一帧高频网格行中。在相邻两个事件或状态突变的时间戳之间，当前指示操作的砖块起始位置（SRT）和目标放置位置（DST）的几何坐标保持恒定。

### 3. 全量列字段字典规约（名称、物理意义与数据源）

#### A. 基础时空上下文信息

* **`Timestamp` (float64)**：**主键。** 来自于 `table/gazepoints.csv` 与 `screen/gazepoints.csv` 原始时间戳的严格并集（Union） 。

* **`subject_id` (string/category)**：**参与者唯一标识符。** 解析自受试者根目录名称（例如 `"69907732"`），用于下游 **LOSO（留一对象交叉验证）** 的数据集划分 。

* **`task` (string/category)**：**装配任务类型。** 解析自子文件夹名称（取值严格限定在 `['car', 'house', 'sc', 'tb', 'tc', 'tsb']` 之中） 。

* **`event` (string)**：**毫秒级离散动作事件。** 来自于 `table/events.csv` 的原始日志，取值为 `['start', 'next', 'previous', 'end']` 之一，通过前向填充（`ffill`）在时间轴上广播。
* **`slide_id` (int64)**：**当前图纸幻灯片页码。** 来自于 `screen/states.csv` 的实时记录，指示当前步骤，通过前向填充（`ffill`）广播。
* **`block_id` (int64)**：**当前理应操作的积木ID。** 属于动态衍生列。通过当前的 `slide_id` 作为索引，实时检索元数据文件 `setup/instructions_{task}.csv` 自动映射获取。

#### B. 动态任务锚点坐标（单位：cm / 区分 4 个角落）

为了精确量化受试者目光与任务目标之间的非线性几何偏差，系统动态引入当前操作积木的物理边界坐标。通过 `instructions_{task}.csv` 提取该积木的 4 个角落，并严格使用后缀 `0, 1, 2, 3` 分别标准化映射到：**`0`=左上、`1`=右上、`2`=右下、`3`=左下**。

* **`target_x_srt_cm0~3` / `target_y_srt_cm0~3` (float64)**：当前操作积木在初始抓取区（Source）的 4 个角落物理坐标（单位：cm，原始的是相对位置，请进行量纲转换）。
* **`target_x_dst_cm0~3` / `target_y_dst_cm0~3` (float64)**：当前操作积木在最终装配区（Destination）的 4 个角落物理坐标（单位：cm，原始的是相对位置，请进行量纲转换）。
* **`target_x_screen_cm0~3` / `target_y_screen_cm0~3` (float64)**：当前 `slide_id` 对应的屏幕指导图纸兴趣区域（AOI）的 4 个角落物理坐标（单位：cm，原始的是相对位置，请进行量纲转换）。通过读取 `setup/slides_{task}.csv` 并将其像素坐标乘以显示器物理尺寸比例转换获得.
* **`target_x_screen_px0~3` / `target_y_screen_px0~3` (float64)**：当前 `slide_id` 对应的屏幕指导图纸兴趣区域（AOI）的 4 个角落像素坐标（单位：px）。通过将上述物理厘米坐标乘以显示器的像素密度（2560/52.7 和 1440/29.6）转换获得.



#### C. 多模态实际注视点坐标（归一化与量纲对齐）

眼动仪输出的原始注视点通常为相对比例或传感器像素值，代码提取时必须统一转换为统一量纲的真实物理度量：

* **`gaze_x_table_cm` / `gaze_y_table_cm` (float64)**：物理厘米单位的 **LEGO 装配表面注视点** 。

$$\text{计算方法：原始相对桌面坐标} \times \text{LEGO 装配表面物理尺寸 } (76 \times 38\text{ cm})$$


* **`gaze_x_screen_cm` / `gaze_y_screen_cm` (float64)**：物理厘米单位的 **屏幕注视点** 。



$$\text{计算方法：原始归一化屏幕坐标} \times \text{显示器物理尺寸 } (52.7 \times 29.6\text{ cm})$$


* **`gaze_x_screen_px` / `gaze_y_screen_px` (float64)**：像素单位的屏幕注视点，用于基线可视化与像素级分析。

$$\text{计算方法：} gaze\_x\_screen\_cm \times \frac{2560}{52.7} \,,\quad gaze\_y\_screen\_cm \times \frac{1440}{29.6}$$



#### D. 论文核心高级衍生标签

* **`active_target_type` (string/category)**：**动态认知焦点指示标签。** 取值范围为 `['SCREEN', 'SRT', 'DST', 'BLINK']` 。用于指示当前毫秒受试者理应关注的物理或图形区域，作为注意力指纹计算的路由开关 。


* **`is_success` (int64)**：**行为结果金标准标签。** 取值为 `[0, 1]` 。这是一个片段级（Segment-level）的前瞻性结果指标。它代表当前 `block_id` 对应的这一组、当前步骤的操作片段，在宏观行为结果上最终是成功了还是失败了（定义见后文逻辑三），用于彻底消除模型训练中的“循环论证”隐患 。

## 三、 核心流水线算法与动态处理逻辑 (Core Advanced Process Logic)

在编写脚本时，必须严格实现以下三大底层计算模块：

### 逻辑一：时间戳并集对齐与坐标 NaN 绝对保留 (Union Timeline & Saccade-Preserving NaN Processing)

人类的眼球运动（尤其是扫视 Saccade）是高度弹道式的、急速跳跃的非线性过程。传统的深度学习流水线通常使用线性插值来补全缺失帧，但这会在两个真实的注视点之间凭空制造出一段不存在的“匀速缓慢滑行”虚假轨迹，破坏专注力指纹（ADF）的物理真实性。

1. **时间轴并集构建**：代码绝不能采用固定的重采样网格（如 60Hz 强行划分）。必须将 `table/gazepoints.csv` 的所有原始时间戳与 `screen/gazepoints.csv` 的所有原始时间戳进行**求并集（Outer Join / Union）**，以此并集长轴作为 Master DataFrame 的基础时间轴。
2. **拒绝坐标插值，保留结构性空值**：如果某一个并集时间戳只有桌面端眼动仪有数据（说明受试者此时低头看手部），那么该行对应的 `gaze_*_screen_*` 字段必须**直接保持为 `NaN`（空值）**；反之，若只有屏幕端有数据，则桌面端坐标保持为 `NaN`。严禁调用任何线性、样条或均值插值函数。
3. **控制信号前向填充**：与坐标字段不同，属于离散宏观上下文的控制信号（如 `event`, `slide_id`, `block_id`）在遇到下一个新事件突变前，其业务状态在物理上是持续生效的。因此，这些离散列必须采用 **前向填充（Forward-fill / `ffill`）** 深度向下复制，直至被新记录刷新。

### 逻辑二：认知焦点状态机有限状态划分 (`active_target_type` Finite State Machine)

在对齐后的每一帧时间戳上，代码必须根据以下规则，动态判定受试者在当前毫秒理应激活的视线比对锚点：

* **状态 1：`SCREEN` (看图纸阶段)**
* **判定条件**：当屏幕眼动仪数据 `gaze_x_screen_cm` 捕获到有效值（非 NaN），或者当前时间戳距离上一次幻灯片切换（`event == 'next'` 或 `'previous'` 触发瞬间）的绝对时差在 2 秒以内（代表切换步骤后受试者本能地抬头阅读图纸说明）。
* **计算效能指向**：该状态下，下游指纹模块将重点计算目光与屏幕上图纸 AOI（`target_*_screen`）的几何演变特征。


* **状态 2：`SRT` (寻砖抓取阶段)**
* **判定条件**：受试者低头，桌面眼动仪数据 `gaze_x_table_cm` 恢复为有效值（非 NaN），且动作日志指示当前 `block_id` **尚未被拿起**（即当前时间戳位于幻灯片页码切换之后，但尚未触发 `pick_up` 或 `grasp` 离散事件之前）。
* **计算效能指向**：此时受试者正在工作台上寻找目标积木，指纹模块将计算目光与积木初始存放区 `target_*_srt` 的空间偏差模式。


* **状态 3：`DST` (对准装配阶段)**
* **判定条件**：一旦 `events.csv` 中捕获到当前 `block_id` 的 `pick_up`、`grasp_success` 或 `assemble_start` 动作事件戳，状态机立即无延迟跳转至 `DST` 状态 。

* **计算效能指向**：此时积木已在受试者手中，其目光锚点正式且唯一地切换到最终拼装目标区 `target_*_dst`，用于量化受试者在执行“精准对准”这一 Safety-Critical（安全关键）操作时的动态波形 。直到点击 `next` 进入下一页幻灯片，状态机重置。




* **状态 4：`BLINK` (异常丢失/眨眼阶段)**
* **判定条件**：当两台眼动仪的坐标同时表现为 `NaN`，且持续时间在合理的生理范围内（如 $100\text{ms} \sim 500\text{ms}$），直接标记为 `BLINK`，后续计算剔除该时段。



### 逻辑三：片段级前瞻标签生成与事故风险逆向广播 (`is_success` Segment-level Back-propagation)

为了彻底封死审稿人关于“循环论证”的质疑，代码不能根据眼动的实时物理距离去盲目定义分心，而是必须通过“最终的行为结果（客观事实）”来反推并定义当前这一段操作片段的认知专注状态：

1. **高频时间轴片段化（Segmentation）**：代码以 `block_id` 的数值切换作为严格的物理边界，将整条连续的高频时间轴切分成一段段互不重叠的“当前步骤操作片段”（Operation Segment）。
2. **结果前瞻性检索（Lookahead Query）**：针对任意一个独立的操作片段，脚本必须向前看（Lookahead），在 `events.csv` 日志中全面检索该片段所覆盖的整个时间窗口内的所有动作结果：
* **判定为 `1` (成功/黄金专注样本)**：若该片段在其生命周期的尾端，成功触发地将当前指示的积木放置到正确位置（可以通过`participant_id\table\state.csv`(注意：需要选择当前步骤结束后所对应的第一个时间戳对应的一行数据)和`setup\instructions_{task}.csv`中的对应坐标位置进行对比），且随后幻灯片页面正常自然地切换至下一页（`event == 'next'`），则判定该步骤操作完美成功，当前片段内**所有高频行**的 `is_success` 均强制填充为 `1` 。
* **判定为 `0` (失败/分心样本)**：若该片段在其生命周期的尾端，操作者没有按照指示将积木放置到正确位置（可以通过`participant_id\table\state.csv`(注意：需要选择当前步骤结束后所对应的第一个时间戳对应的一行数据)和`setup\instructions_{task}.csv`中的对应坐标位置进行对比），则判定该步骤操作失败，当前片段内**所有高频行**的 `is_success` 均强制填充为 `0` 。


3. **逆向状态广播**：判定结果出来后，将该二进制数值（`0` 或 `1`）逆向广播（Broadcast）填充给该片段内包含的每一帧高频眼动时间戳行。从而保证 downstream 模型在读取任意一帧视线波形时，都能拥有底层行为结果的标签支撑.

## 四、 工业级代码质量与边缘案例防坑指南 (Code Quality & Optimization Specs)

* **内存友好型显式下采样与数据类声明**：由于 GAIPAT 包含 80 个受试者的多模态高频时序数据，整体合并后矩阵极其庞大 。代码中必须将 `subject_id`, `task`, `active_target_type`, `event` 显式转换为 `category` 数据类型存储。坐标字段（`gaze_*`, `target_*`）一律下采样使用 `float32`，标签字段（`is_success`, `slide_id`）使用 `int8` / `int16`。严禁全部默认使用 `float64` 以防止大矩阵引发内存溢出（OOM）。


* **NaN 值的工程哲学**：如前文所述，`NaN` 是人类在空间中切换视线或产生眨眼生理现象的自然物理表现。代码中严禁盲目调用 `fillna(method='bfill')` 或 `dropna()`。缺失值必须被视作有价值的特征予以保留，在后续喂给 DTW（动态时间规整）计算时，由 DTW 层的输入解析器动态剔除包含 `NaN` 的非桌面帧即可 。


* **完全向量化（Vectorization）拒绝 Python 原生循环**：由于单任务数据行数达百万级，在实现逻辑二的状态机切换和逻辑三的片段级标签填充时，**严禁**使用 `for index, row in df.iterrows():` 这种效率极低的显式循环。必须完全基于 `pandas` 的 `groupby().transform()`，或者 `numpy.select(condlist, choicelist)` 等高性能向量化掩码操作来完成整表的并行化计算，确保整个清洗流程在数分钟内运行完毕。
